# M4 Target-Conditioned Arm Env Design

**Status**: Draft v0

Env id:

```text
TargetConditionedArmGrasp-v0
```

Purpose: Month 2 entry point for single-arm target-conditioned manipulation. This env keeps the
same right-arm joint-delta controller as `StaticArmGrasp-v0`, but adds target/distractor semantics
needed by BC/RL.

## Scene

- Static base.
- Right arm only.
- Three tabletop cubes: red, blue, green.
- One target color is sampled per episode unless `target_color` is fixed.
- Remaining cubes are distractors.

## Action

Same as `StaticArmGrasp-v0`:

```text
right arm 6D joint delta = 5 arm joints + gripper
```

## Extra Observation

The env exposes sim-GT target conditioning fields for early BC/RL plumbing:

| Field | Shape | Meaning |
|-------|-------|---------|
| `target_pos_base` | `(N, 3)` | Target cube position in base/world frame. |
| `target_color_id` | `(N, 1)` | `red=0`, `blue=1`, `green=2`. |
| `skill_id` | `(N, 1)` | `top_grasp=0` for M4 v0. |
| `distractor_pos_base` | `(N, 6)` | Two distractor cube positions flattened as `[x1, y1, z1, x2, y2, z2]`. |
| `distractor_color_ids` | `(N, 2)` | Distractor color IDs. |
| `tcp_to_target` | `(N, 3)` | Target position minus TCP position. |

Visual masks are not generated by this env. M1/perception owns `target_mask` and `distractor_mask`.
For early sim training, these GT positions are enough to wire target-conditioned reward and policy
inputs.

## Evaluation

Success is target-specific:

```text
success = target grasped and target lifted > 5cm for 10 steps and no distractor lifted
```

Reported metrics:

| Metric | Meaning |
|--------|---------|
| `target_is_grasped` | Target cube is grasped by the gripper. |
| `target_lift_height` | Target lift height above table. |
| `wrong_object_grasped` | Any distractor is grasped. |
| `wrong_object_lifted` | Any distractor is lifted over threshold. |
| `wrong_object_failure` | Alias of `wrong_object_lifted`. |
| `max_distractor_lift_height` | Largest distractor lift height. |

## Reward v0

```text
r = -0.5 * distance(tcp, target)
    + 1.0 * target_is_grasped
    + 5.0 * target_lift_height_positive
    + 10.0 * success
    - 5.0 * wrong_object_lifted
    - 1.0 * wrong_object_grasped
    - 0.001 * action_norm
```

This is intentionally simple. It is meant to establish the target-conditioned interface before
BC initialization, reward classifiers, or PPO/SAC fine-tuning.

## Known Limits

- Target color is one scalar per vectorized reset, not per parallel sub-env.
- Uses sim-GT positions, not M1 masks, for early training plumbing.
- Does not implement EE-space delta yet; current controller remains joint-delta.
- Does not include BC dataset loading or policy code.
